  <acceptance_criteria>
    Every checklist item below must be true. If any fails, describe the specific failure so Plan 02 can be revised.

    **Spin flow (CORE-01, CORE-02, CORE-03):**
    - [ ] A single tap on "Spin" triggers the animation within 100ms of click
    - [ ] Within ~1.5s, a result card appears showing: name (large), description (smaller), price ("{N} kr")
    - [ ] The button label changes to "Spin again" after the first result
    - [ ] Clicking "Spin again" produces a NEW result (try at least 3 spins — they should not all be identical)
    - [ ] No page reload happens at any point

    **Filters (FILT-01, FILT-02):**
    - [ ] Three pills visible: All / Veg / Meat
    - [ ] "All" is active (highlighted dark) on first load
    - [ ] Clicking "Veg" updates the count to 6 pizzas available (6 vegetarian in menu.json)
    - [ ] Clicking "Meat" updates the count to 8 pizzas available (8 meat in menu.json)
    - [ ] Clicking "All" restores count to 14
    - [ ] Switching filter while a result is showing CLEARS the result card (Pitfall 5)
    - [ ] Spinning under "Veg" filter produces only pizzas with 🌱 Vegetarian badge
    - [ ] Spinning under "Meat" filter produces only pizzas with 🥩 Meat badge

    **Disabled state (FILT-03):**
    - [ ] The `{#if filteredPizzas.length === 0}` branch is present in `src/components/PizzaPicker.svelte`
    - [ ] If live-tested with empty menu: Spin button visually differs (bg-neutral-300, not-allowed cursor) and is not clickable
    - [ ] If live-tested: Explanation text "No pizzas match this filter" + "Switch to All or try a different filter." is visible

    **Animation (D-01, D-02, D-03):**
    - [ ] The pizza emoji 🍕 rotates smoothly during the spin
    - [ ] Animation feels ease-out (fast start, decelerates to a stop) — not linear, not bouncy
    - [ ] Duration feels like ~1.5 seconds — not rushed, not sluggish
    - [ ] No JavaScript animation library is shipped (Network tab: no motion/gsap/framer-motion)

    **Layout (D-06, D-10, D-12):**
    - [ ] Desktop (>400px wide): picker is centered horizontally AND vertically in the viewport with max-width ~400px
    - [ ] Mobile (≤400px, or devtools mobile emulation): picker fills the viewport width with internal padding
    - [ ] Top-to-bottom order: heading → subline → filter pills → count → emoji + Spin button → (result card when present)

    **Visual (D-04, D-05, D-07, D-11):**
    - [ ] Pizza name on the result card is the largest, most dominant text (24px semibold)
    - [ ] Description is smaller and muted (14px neutral-500)
    - [ ] Price is subtle (14px)
    - [ ] Diet badge shows 🌱 Vegetarian (green) for vegetarian pizzas, 🥩 Meat (orange) for meat pizzas
    - [ ] Filter pills are rounded, have visible active state (dark fill), touch-friendly (~48px min-height)
    - [ ] Palette is clean/neutral — NO warm red/orange/yellow on background or buttons (only on diet badges)
    - [ ] Accent color (dark) appears ONLY on: active filter pill, Spin button background (when enabled), hover states

    **Accessibility (UI-SPEC Accessibility section):**
    - [ ] Result card container has `role="status"` and `aria-live="polite"` (devtools inspector)
    - [ ] Filter pill buttons have `aria-pressed` reflecting state (devtools inspector)
    - [ ] Pizza emoji span has `aria-hidden="true"` (devtools inspector)

    **Copywriting (UI-SPEC Copywriting Contract):**
    - [ ] Page heading reads: "ZZ Pizza Picker"
    - [ ] Subline reads: "One tap. One pizza. No regrets."
    - [ ] Count format: "{N} pizzas available"
    - [ ] Button labels: "Spin" → "Spinning…" → "Spin again"
    - [ ] Page `<title>` is "ZZ Pizza Picker"
  </acceptance_criteria>
