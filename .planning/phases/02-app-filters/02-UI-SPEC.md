# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: App & Filters.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-rolled with Tailwind utilities |
| Icon library | none — emoji only (🍕 🌱 🥩) |
| Font | system-ui, -apple-system, sans-serif (system stack) |

**Rationale:** This is an Astro 6 + Svelte 5 project with Tailwind CSS v4. shadcn/ui is React-only and does not apply. No component registry is used. All UI elements are built directly with Tailwind utility classes inside `PizzaPicker.svelte`.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between filter pills, icon-text gaps |
| sm | 8px | Padding inside filter pills (vertical), gap between spin button and count text |
| md | 16px | Padding inside filter pills (horizontal), result card internal padding |
| lg | 24px | Vertical gap between major UI sections (filters → spin → result) |
| xl | 32px | Vertical padding above/below the full picker island |
| 2xl | 48px | Top/bottom viewport centering inset on desktop |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets for filter pills must be minimum 44px tall (accessibility — iOS guideline). Achieved by combining `py-2` (8px) + `text-sm` line height, plus explicit `min-h-[44px]` if needed.

## Typography

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Body / description | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-relaxed` |
| Label / count / badge | 14px | 500 (medium) | 1.4 | `text-sm font-medium leading-snug` |
| Heading / pizza name | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` |
| Display / spin emoji | 48px | n/a (emoji) | 1.0 | `text-5xl` |

**Font stack (apply to root `body`):**
```
font-family: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, sans-serif;
```
Source: CONTEXT.md D-11 (system font stack is fine) and RESEARCH.md (Claude's discretion).

**Constraints:**
- Exactly 3 text sizes in use: 14px, 24px, 48px. The 14px Label role covers both body description text and UI labels to avoid a fourth size.
- Exactly 2 weights: 400 (regular) for all body/description text; 600 (semibold) for pizza name heading. Filter pill labels use 500 (medium) as a mid-level label weight.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafafa` (neutral-50) | Page background — white/off-white per D-11 |
| Secondary (30%) | `#ffffff` (white) | Result card surface, elevated from page background |
| Accent (10%) | `#171717` (neutral-900) | Active filter pill background + text inversion; spin button background |
| Muted text | `#737373` (neutral-500) | Description text, count text, disabled explanation text |
| Border | `#e5e5e5` (neutral-200) | Result card border, inactive pill border |
| Destructive | Not used in this phase | — |

**Accent reserved for:**
1. Active filter pill — background becomes `neutral-900`, text becomes `white`
2. Spin button — background `neutral-900`, text `white`
3. Spin button hover state — background `neutral-700`

Accent is NOT used on: the result card, the pizza name, body text, price, or the disabled state. Inactive filter pills use `neutral-100` fill with `neutral-700` text — neutral, not accented.

**Disabled state colors (FILT-03):**
- Spin button disabled: `bg-neutral-300 text-neutral-400 cursor-not-allowed`
- Explanation text: `text-neutral-500` (muted, not alarming)

**Veg/Meat badge colors:**
- Vegetarian badge: `bg-green-50 text-green-700` — soft green, not primary accent
- Meat badge: `bg-orange-50 text-orange-700` — soft orange, not primary accent
- These semantic colors are badge-only; they never appear on interactive controls.

Source: CONTEXT.md D-11 (clean minimal, no warm pizza colors), D-12 (neutral accent approach).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page title (`<title>`) | ZZ Pizza Picker |
| App heading (visible H1) | ZZ Pizza Picker |
| App subline | One tap. One pizza. No regrets. |
| Filter pill — All | All |
| Filter pill — Vegetarian | Veg |
| Filter pill — Meat | Meat |
| Pizza count (FILT-02) | {N} pizzas available — placed directly below filter pills, above spin button |
| Primary CTA — ready | Spin |
| Primary CTA — spinning | Spinning… |
| Result card — diet badge (veg) | 🌱 Vegetarian |
| Result card — diet badge (meat) | 🥩 Meat |
| Result card — price | {price} kr |
| Re-spin CTA (replaces Spin after result) | Spin again |
| Empty state heading (FILT-03) | No pizzas match this filter |
| Empty state body | Switch to All or try a different filter. |
| Spin button disabled label | Spin (aria-label: "Spin disabled — no pizzas match this filter") |

**Pizza count placement decision (FILT-02, Claude's discretion from CONTEXT.md D-09):**
Place the count as a single line `"{N} pizzas available"` between the filter pills and the spin button. This keeps it at the decision point — the user sees the pool size before committing to a spin. At `text-sm text-neutral-500`, it reads as supporting information without competing with the CTA.

**No destructive actions in this phase.** No confirmation dialogs needed.

**Error states (data load failure):**
If `menu.json` fails to parse or the pizza array is empty at runtime (unexpected — Phase 1 guards against this at build time):
- Heading: Something went wrong
- Body: The menu couldn't load. Try refreshing the page.

## Component Inventory

One Svelte 5 component handles the entire interactive island.

| Component | File | Rendered by |
|-----------|------|-------------|
| `PizzaPicker` | `src/components/PizzaPicker.svelte` | `src/pages/index.astro` via `client:load` |

**No sub-components.** The component is ~100–150 lines and splitting it would add navigation cost with no readability gain. Source: RESEARCH.md (single component recommendation).

```
  <h1>             — App heading

```

## Interaction States

### Spin Button

| State | Appearance | Trigger |
|-------|-----------|---------|
| Ready | `bg-neutral-900 text-white` — full opacity | `filteredPizzas.length > 0 && !spinning` |
| Spinning | `bg-neutral-900 text-white opacity-75` + label "Spinning…" | `spinning === true` — also disabled during animation |
| Disabled (no pizzas) | `bg-neutral-300 text-neutral-400 cursor-not-allowed` | `filteredPizzas.length === 0` |
| Post-result | Label changes to "Spin again" — same styling as Ready | `result !== null && !spinning` |

### Filter Pills

| State | Appearance |
|-------|-----------|
| Inactive | `bg-neutral-100 text-neutral-700 border border-neutral-200` |
| Active | `bg-neutral-900 text-white border-transparent` |
| Hover (inactive) | `bg-neutral-200` |

### Result Card

| State | Appearance | Trigger |
|-------|-----------|---------|
| Hidden | Not rendered (no DOM node) | `result === null` |
| Visible | Fades in with `opacity-0 → opacity-100` CSS transition (200ms) | `result !== null` |
| Cleared | Returns to hidden | Filter changes — `$effect` clears `result` on `activeFilter` change |

### Pizza Emoji Animation

| State | Appearance | Timing |
|-------|-----------|--------|
| Idle | Static 🍕 | n/a |
| Spinning | CSS `@keyframes spin-pizza` rotates 0 → 720deg | 1.5s, `cubic-bezier(0.33, 1, 0.68, 1)` ease-out, `forwards` fill mode |

**Animation detail:** Two full rotations (720deg) with ease-out deceleration. The spinning is the reveal — no skeleton or mystery state. After 1500ms, the `spinning` flag becomes false and the result card appears.

Source: CONTEXT.md D-01, D-02, D-03.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Spin button — spinning state | `aria-label="Spinning, please wait"` during animation |
| Spin button — disabled state | `aria-label="Spin disabled — no pizzas match this filter"` |
| Filter pills — active state | `aria-pressed="true/false"` on each button |
| Pizza emoji | `aria-hidden="true"` — decorative, not informational |
| Result announcement | Result card container gets `role="status" aria-live="polite"` — screen reader announces new result |
| Color contrast | All text meets WCAG AA: neutral-900 on neutral-50 = 19:1; neutral-500 on neutral-50 = 4.6:1 (passes AA for normal text at 14px) |
