**Part A — Connect repo to Cloudflare Pages (DEPL-01, D-06, D-09, D-10):**

1. Go to https://dash.cloudflare.com → log in
2. Left sidebar: Workers & Pages → Create application → Pages → Connect to Git
3. If first time: Authorize the Cloudflare GitHub App (grants CF read access to your repos)
4. Select the `zz-pizza-picker` repository
5. Set build configuration:
   - Framework preset: Astro
   - Build command: `npm run build`
   - Build output directory: `dist`
   - Root directory: (leave blank — uses repo root)
6. Click Save and Deploy
7. Wait for the first build to complete (2-4 minutes — CF runs `npm run build` which triggers
   the `prebuild` scraper, then `astro build`)
8. Note the assigned URL — target is `zz-pizza-picker.pages.dev` (D-10); if that name is
   taken CF assigns a variant — any assigned name is acceptable

**Part B — Create deploy hook (DEPL-02, D-07):**

1. In the CF Pages project (just created) → Settings → Builds & deployments
2. Scroll to Deploy hooks → Add deploy hook
3. Name: `github-actions-weekly`
4. Branch: `main`
5. Click Generate
6. CF provides a URL like:
   `https://api.cloudflare.com/client/v4/pages/webhooks/deploy_hooks/<uuid>`
7. Copy this URL — treat it as a password; do not paste it anywhere except GitHub Secrets

**Part C — Add deploy hook URL as GitHub Secret:**

1. Go to your GitHub repository → Settings → Secrets and variables → Actions
2. Click New repository secret
3. Name: `CF_DEPLOY_HOOK_URL` (exact name — must match `.github/workflows/weekly-rebuild.yml`)
4. Value: paste the CF deploy hook URL from Part B step 7
5. Click Add secret

After all three parts, tell me the assigned Cloudflare Pages URL.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser → Cloudflare Pages CDN | Public site; all traffic is HTTPS (CF enforces TLS by default on pages.dev) |
| GitHub Actions → CF deploy hook | POST from GH runner to CF URL stored in GitHub Secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Information Disclosure | CF deploy hook URL during dashboard setup | mitigate | Copy hook URL directly into GitHub Secrets; do not paste into chat, notes, or any unencrypted storage; Task 1 action block explicitly flags this |
| T-03-07 | Tampering | Unauthorized CF Pages deploys via exposed hook URL | accept | Deploy hook URL is the only credential (no auth header required); risk is limited to triggering rebuilds (not data exfiltration); URL is stored only in GitHub Secrets (restricted access) |
| T-03-08 | Elevation of Privilege | CF GitHub App authorization scope | accept | CF GitHub App requests read-only repo access; cannot write to or modify repo content; standard CF Pages setup path |
</threat_model>
